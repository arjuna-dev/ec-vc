# Chat Acronyms

Shared shorthand for our chats. These terms are meant to be taken literally.

## Chat Commands

These are immediate conversational or execution-control commands.

`Qi`
- Quick iteration.
- Make one small pass at a time and check understanding often.

`NCT`
- Non-Commit Test.
- Trial, inspect, lint, run, or test without creating a commit unless explicitly asked.

`Doc`
- Document decision.
- Capture the rule, conclusion, or architecture decision in writing.

`Stop`
- Stop immediately.
- No further edits, no more tool actions, no continued implementation.

## Work Modes

These shape how the work should be approached.

`Review`
- Inspect and find issues.
- Do not implement unless explicitly asked.

`Plan`
- Define sequence and structure before action.
- Analyze first, propose order of operations, do not jump into code.

`Lock_Scope`
- Do only `[this]` and nothing else.
- No side cleanup, no adjacent fixes, no reinterpretation.

`STRICT_DISCOVERY`
- Treat the task as architecture and contract discovery work.
- Expose breakage and missing contracts instead of smoothing them over.

`POLISH`
- Treat the task as a refinement pass.
- Improve clarity, consistency, usability, and finish within the requested scope.

`DOCUMENT`
- Treat the task as documentation work.
- Prioritize writing and clarifying decisions, rules, and guidance.

`MIGRATION`
- Treat the task as intentional structure or ownership migration work.
- Prioritize canonical ownership, strict sequencing, and safe transitions.

## Working Notes

- If a term above is used in chat, interpret it literally.
- Chat Commands affect immediate behavior.
- Work Modes affect the approach and decision style for the task.
- If a term is unclear, ask before acting.
- New acronyms can be added here as our shared language evolves.
