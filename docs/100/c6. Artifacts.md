## Purpose

`Artifacts` stores raw source artifacts and their primary reference metadata.

It exists to hold source materials before downstream processing, extraction, and created-file lineage.

## Glossary

| Term | Meaning |
| --- | --- |
| `Artifact` | A source file or source material record. |
| `Raw` | The original source state. |
| `Provenance` | The record of who added the artifact and how it entered the system. |
| `Created Files` | Downstream files born from ingestion or extraction. |

## Operating Rules

- Treat `Artifacts` as the raw source file layer.
- Preserve provenance from the acting user through system fields.
- Do not let downstream assumptions overwrite source truth silently.
